# 基于 AI 辅助编程的无线通信文件传输基带仿真系统

## 摘要

本项目构建了一个完整的无线通信基带仿真系统，实现文本文件通过模拟无线链路端到端传输。系统采用 QPSK 调制（Gray 映射）结合 (2,1,7) 卷积码与 Viterbi 译码，使用 32-bit PN 序列进行帧同步，信道建模为 AWGN。发射链路依次完成信源编码、LFSR 自同步加扰、卷积编码、帧封装和 QPSK 调制；接收链路通过滑动相关帧同步、QPSK 硬解调、Viterbi 译码、解扰和信源解码恢复原始文件。系统提供统一 CLI 入口，支持 SNR、seed、同步偏移、帧长等参数配置。测试结果表明：在 SNR≥12dB 时 BER 和 FER 均为 0，文本恢复率 100%；SNR=10dB 时 BER=0、FER≈0.18，体现了卷积码在 AWGN 信道下的编码增益。系统共编写 109 个单元与集成测试，全部通过。

**关键词**：无线通信；QPSK；AWGN；帧同步；信道编码；Viterbi 译码；AI 辅助编程

## 1 项目概述

### 1.1 项目背景与目标

无线通信系统的核心任务是在不可靠的物理信道上实现可靠的数据传输。本项目来源于《无线通信技术》课程期末项目，要求构建一个基带仿真系统，将文本文件 `Test.txt` 经过完整的无线通信基带处理链路传输，并在接收端恢复为 `received.txt`。

系统需要模拟真实无线通信中的关键环节：信源编码将文本转换为比特流，加扰器消除长连 0/1 以辅助接收端定时恢复，信道编码添加冗余比特对抗信道噪声，帧封装为数据添加同步标记，QPSK 调制将比特映射为复基带符号，AWGN 信道模拟自由空间中的热噪声，帧同步在接收符号流中定位帧边界，最后通过解调、译码、解扰和解码恢复原始文件。

项目的验收标准包括：统一命令行入口、支持不同 SNR/seed/同步偏移参数、无硬编码行为、输出 `received.txt` 和 `metrics.json` 文件，以及生成星座图和 BER 曲线。

### 1.2 PRD 关键需求理解

根据 PRD 要求，系统设计遵循以下核心约束：

1. **固定系统链路**：信源编码→加扰→信道编码→帧封装→QPSK 调制→AWGN→帧同步→QPSK 解调→信道译码→解扰→信源解码，模块顺序不可变更。
2. **统一命令行接口**：通过 `python -m src.cli` 启动，支持 `--input`、`--output-dir`、`--snr`、`--seed`、`--sync-offset`、`--frame-length` 等参数。
3. **必需输出文件**：`received.txt`（恢复的文本文件）、`metrics.json`（BER/FER/文本恢复率等指标）。
4. **验收标准**：支持不同文本、SNR、seed、同步偏移下的测试，无硬编码绕过链路的行为。
5. **AI 使用记录**：完整记录 AI 辅助编程过程于 `AI_LOG.md`。

| Viterbi 译码 | `channel_coder.py` | coded: ndarray[uint8] | decoded: ndarray[uint8] |

### 2.3 帧结构设计

```
┌──────────────────┬──────────────────┬─────────────────────────┐
│  Sync Word       │  Length          │  Payload                │
│  32 bits         │  16 bits         │  N bits (默认 256)      │
│  (PN 序列)       │  (大端序无符号)   │  (编码后数据)            │
└──────────────────┴──────────────────┴─────────────────────────┘
                       ▲
           帧总长 = 32 + 16 + 256 = 304 bits = 152 QPSK 符号
```

- **同步字**：32-bit PN 序列 `11101110 01011001 10100111 01100001`，具有良好的非周期自相关特性
- **长度字段**：16-bit 大端序无符号整数，指示本帧有效载荷比特数（≤256）
- **载荷**：不足一帧时零填充

### 2.4 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 调制方式 | QPSK (Gray 映射) | bit0→I, bit1→Q |
| 信道编码 | 卷积码 (2,1,7) | G1=171₈, G2=133₈, 码率 1/2 |
| 自由距离 | 10 | 理论纠错能力 |
| 译码方式 | Viterbi MLSE | 硬判决（汉明距离）+ 软判决（欧氏距离） |
| 同步方式 | 滑动相关 + 峰值检测 | 按相关值降序排列后筛选 |
| 扰码多项式 | x⁷ + x⁴ + 1 | 自同步 LFSR |
| 帧长 | 304 bits (152 symbols) | 可配置 |

## 3 系统实现

### 3.1 信源编解码

使用 Python `numpy.packbits` / `numpy.unpackbits` 实现字节与比特的高效转换，支持 UTF-8 编码以处理中、日、韩等多语言文本。metadata 字典携带原始文件名和比特数，确保接收端精确恢复文件大小。

### 3.2 加扰器

实现基于线性反馈移位寄存器（LFSR）的自同步加扰器。生成多项式为 x⁷+x⁴+1（CCITT 标准），初始化状态为全 1（0x7F）。自同步设计确保解扰器无需显式同步——即使初始状态不同，经过 7 个比特后自动收敛。

### 3.3 卷积编码与 Viterbi 译码

**编码器**：(2,1,7) 卷积码，生成多项式 G1=171₈（二进制 1111001）、G2=133₈（二进制 1011011），码率 1/2。采用零尾比特终止（追加 K-1=6 个零），驱动编码器回到全零状态。

**Viterbi 译码器**：实现 64 状态（2⁶）网格搜索。关键技术决策：
- 存储"前一状态"而非"输入比特"作为幸存路径信息——因为卷积码的状态转移不可逆（两个不同的前一状态可能在相同输入比特下映射到同一后继状态）
- 软判决路径使用 `argmin(path_metric[-1])` 而非硬编码状态 0，确保有噪声时的最大似然终止状态估计

### 3.4 帧封装

帧封装器将编码后的比特流组织为固定长度帧。每帧包含 32-bit 同步字、16-bit 长度字段（大端序）和最多 256-bit 载荷。最后一帧不足时零填充，接收端根据长度字段精确截断。

### 3.5 QPSK 调制解调

采用 Gray 映射 QPSK：bit0 控制 I 路（0→+1, 1→-1），bit1 控制 Q 路（0→+1, 1→-1）。星座点归一化至平均功率 1。解调器同时提供硬判决（直接最近邻）和软判决（LLR 对数似然比）两种模式。

### 3.6 AWGN 信道

叠加复高斯白噪声：n ~ CN(0, N₀)，其中 N₀ = 10^(-SNR_dB/10)。支持 seed 参数确保噪声可复现。噪声生成使用 numpy RandomState 以保证确定性。

### 3.7 帧同步

实现归一化滑动互相关检测器。关键设计决策：
- **按相关值降序排列峰值**：确保最强（真实）峰值优先保留，避免短同步字的随机误检遮挡真实帧起始位置
- **最小帧间距**：帧长减去同步字长，防止同一帧内重复检测
- **归一化**：按参考信号和接收信号段的功率归一化，使阈值因子（默认 0.5）在不同 SNR 下保持一致

### 4.2 SNR 扫描测试

使用 `scripts/sweep_snr.py` 对 Test.txt（768 字节）进行 SNR -2 ~ 12 dB 范围扫描：

| SNR (dB) | BER | FER | 说明 |
|----------|-----|-----|------|
| -2 | 0.5007 | 1.0000 | 噪声过大，完全无法通信 |
| 0 | 0.4855 | 1.0000 | BER≈0.5，等效随机猜测 |
| 2 | 0.4665 | 1.0000 | 略低于 0.5（编码提供微弱增益） |
| 4 | 0.4578 | 1.0000 | 帧同步全部失败 |
| 6 | 0.4564 | 1.0000 | 同上 |
| 8 | 0.2982 | 0.8776 | 开始出现可恢复帧 |
| **9** | **0.0000** | 0.5714 | **瀑布区入口：BER 清零** |
| **10** | **0.0000** | 0.1837 | 大部分帧成功恢复 |
| **11** | **0.0000** | 0.0408 | 仅极少数帧失败 |
| **12** | **0.0000** | **0.0000** | **完全可靠通信** |

### 4.3 结果分析

**瀑布效应**：卷积码在 SNR ≈ 9dB 处表现出典型的阈值行为。SNR < 9dB 时 BER 接近 0.5（等效随机），SNR ≥ 9dB 时 BER 骤降至 0。这体现了 Viterbi 译码的非线性特性——一旦信道质量超过译码器的纠错能力阈值，性能急剧改善。

**FER vs BER 分离**：在 SNR 9-11dB 区间，BER=0 但 FER>0。原因在于：即使译码器能完美纠正载荷比特错误，个别帧的长度字段（16-bit 无编码保护）可能被噪声破坏，导致该帧被判定为"帧错误"。这是当前帧结构的一个设计局限。

**编码增益**：对比未编码 QPSK 的理论 BER 曲线（在 10⁻⁵ 处约需 9.6dB Eb/N₀），本系统的 (2,1,7) 卷积码在 SNR=10dB 时已实现 BER=0，体现了约 3-4 dB 的编码增益。

### 4.4 输出文件验证

- **received.txt**：SNR≥12dB 时与 Test.txt 逐字符完全一致（文本恢复率 100%）
- **metrics.json**：包含 BER、FER、text_recovery_rate、total_bits、bit_errors、total_frames、frame_errors、snr_db、seed、sync_offset 等完整指标
- **constellation_rx.png**：接收星座图清晰展示 AWGN 噪声对 QPSK 星座点的扩散效果
- **sync_correlation.png**：滑动相关曲线在帧起始位置出现尖锐峰值
- **ber_vs_snr.png**：BER 和 FER 曲线直观展示瀑布效应
